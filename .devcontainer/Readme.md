# .devcontainer/config の使い方

- `append/` と `replace/` にホームディレクトリ相対のパスでファイルを置くと、自動で `~/` 以下に適用されます（例: `.bashrc`, `.config/git/config` など）。
- `replace/` 配下は対象ファイルをそのまま上書きします。既存ファイルと内容が同一なら何も変更しません。
- `append/` 配下は対象ファイルの末尾にセンチネル付きで差し込みます。再実行時は同じセンチネルブロックを置き換えるため重複しません。
- フォルダ階層もそのまま反映されます。`append/.config/foo/bar` は `~/.config/foo/bar` に適用されます。
- 設定ファイルを追加・変更した後は `bash .devcontainer/hooks/apply-configs.sh` を実行すると新しい内容が反映されます。

## 動作ルール
- 対象のホーム側ファイルが存在しない場合  
  - `replace/`: ディレクトリを作成し、ファイルを新規作成して中身をコピーします。  
  - `append/`: ディレクトリを作成し、センチネル付きブロックを新規追加する形で作成します。
- 同じ相対パスが `replace/` と `append/` の両方にある場合  
  - `replace` が先に反映され、続いて `append` が末尾にセンチネル付きで差し込まれます（`replace` の内容を基に末尾追加されるイメージ）。
